01 — Introduction

Go 언어 소개

Go(또는 Golang)는 2009년 구글의 Robert Griesemer, Rob Pike, Ken Thompson에 의해 설계된 오픈소스 프로그래밍 언어입니다. C의 성능과 Python의 생산성을 결합하겠다는 목표로 시작되었으며, 현재 Kubernetes, Docker, Terraform 등 핵심 인프라 프로젝트에 광범위하게 사용됩니다.

⚡
빠른 컴파일

정적 링크와 최적화된 컴파일러로 수백만 줄 코드도 수 초 내에 빌드됩니다.

🔒
정적 타입 시스템

컴파일 타임에 타입 오류를 검출해 런타임 버그를 사전에 방지합니다.

🔄
고루틴 동시성

경량 스레드인 Goroutine으로 수만 개의 동시 작업을 효율적으로 처리합니다.

🗑️
가비지 컬렉션

트라이컬러 마크-스윕 GC로 메모리를 자동 관리, STW(Stop-The-World) 시간을 최소화합니다.

📦
단일 바이너리

외부 런타임 없이 단일 실행 파일로 컴파일됩니다. 배포가 극도로 단순합니다.

🔗
C 상호 운용

CGo를 통해 C 라이브러리를 직접 호출할 수 있어 기존 에코시스템을 활용합니다.

🟢 Go가 잘하는 것

  • 네트워크 서버 / 마이크로서비스
  • CLI 도구 및 시스템 유틸리티
  • 컨테이너 / 클라우드 인프라
  • 고성능 데이터 파이프라인
  • 분산 시스템 구축

🔴 Go가 약한 것

  • GUI 데스크탑 애플리케이션
  • 수치 계산 / ML (Python 대비)
  • 임베디드 / 베어메탈 (Rust 대비)
  • 고급 제네릭 프로그래밍 패턴
  • 브라우저 내 실행 (제한적)

고루틴과 스케줄러 (GMP 모델)

Go의 동시성은 GMP 스케줄러로 구현됩니다. OS 스레드를 직접 사용하는 대신 경량의 고루틴을 M:N 방식으로 스케줄링합니다.

goroutine 1 goroutine 2 goroutine 3 goroutine 4 goroutine 5

GMP 구성 요소

  • G (Goroutine) — 실행 단위, ~2KB 스택
  • M (Machine) — OS 스레드 래퍼
  • P (Processor) — 논리 프로세서, 로컬 런큐 보유
  • Work Stealing으로 부하 분산
  • GOMAXPROCS로 P 수 제어

고루틴 스택

  • 초기 스택 2KB (OS 스레드 ~1MB 대비)
  • 필요 시 동적으로 성장 (최대 1GB)
  • 연속 스택 전략 사용
  • 채널 통신으로 동기화
  • select로 복수 채널 처리

가비지 컬렉터

Go GC는 트라이컬러 마크-스윕(Tri-color Mark and Sweep) 알고리즘을 사용하며 대부분의 작업이 애플리케이션과 병렬로 실행됩니다.

⚪ White (미방문)

아직 방문하지 않은 객체. GC 사이클 완료 후 흰색으로 남은 객체는 수거됩니다.

🔘 Gray (처리 중)

방문했지만 참조 대상을 아직 검사하지 않은 객체. 워크리스트에 대기 중입니다.

⚫ Black (완료)

해당 객체와 모든 참조 대상 검사 완료. 이번 사이클에서 수거되지 않습니다.

💡 메모리 관리 팁

Go에서 변수가 힙에 할당될지 스택에 할당될지는 이스케이프 분석(Escape Analysis)으로 결정됩니다. go build -gcflags="-m" 옵션으로 이스케이프 분석 결과를 확인할 수 있습니다.

package main

import "fmt"

func main() {
    // var 키워드로 명시적 선언
    var name string = "Gopher"
    var age  int    = 13
    var pi   float64 = 3.14159
    var ok   bool   = true

    // := 단축 선언 (함수 내부에서만 사용 가능)
    city := "Seoul"
    score := 98.5

    // 다중 선언
    x, y, z := 1, 2, 3

    // 상수
    const MaxSize = 1024
    const Pi float64 = 3.14159265358979

    fmt.Printf("%s(%d)는 %s에 삽니다. score=%.1f\n", name, age, city, score)
    _ = pi; _ = ok; _ = x; _ = y; _ = z; _ = MaxSize
}

배열, 슬라이스, 맵

collections.go 
package main

import "fmt"

func main() {
    // 배열 — 고정 길이
    var arr [3]int = [3]int{10, 20, 30}
    fmt.Println(arr[0]) // 10

    // 슬라이스 — 동적 배열, Go에서 가장 많이 쓰임
    nums := []int{1, 2, 3, 4, 5}
    nums = append(nums, 6)       // 원소 추가
    sub := nums[1:4]               // 슬라이싱: [2 3 4]
    fmt.Println(sub)

    // make로 슬라이스 생성 (len=0, cap=10)
    s := make([]string, 0, 10)
    s = append(s, "Go", "Rust", "C")

    // 맵 — key-value 자료구조
    scores := map[string]int{
        "Alice": 95,
        "Bob":   82,
    }
    scores["Charlie"] = 91

    // 값 존재 여부 확인 (ok idiom)
    val, ok := scores["Dave"]
    if !ok {
        fmt.Println("Dave 없음, 기본값:", val)
    }
}

조건문과 반복문

control.go 
package main

import "fmt"

func classify(n int) string {
    // if 초기화 구문 지원
    if rem := n % 2; rem == 0 {
        return "짝수"
    } else {
        return "홀수"
    }
}

func main() {
    // Go에는 while이 없다 — for로 통일
    for i := 0; i < 5; i++ {
        fmt.Printf("%d는 %s\n", i, classify(i))
    }

    // while처럼 사용
    n := 1
    for n < 128 {
        n *= 2
    }

    // range로 슬라이스 순회
    fruits := []string{"🍎", "🍊", "🍋"}
    for idx, fruit := range fruits {
        fmt.Printf("[%d] %s\n", idx, fruit)
    }

    // switch — fallthrough 없음이 기본
    day := "토요일"
    switch day {
    case "토요일", "일요일":
        fmt.Println("주말!")
    default:
        fmt.Println("평일")
    }
}

함수

functions.go 
package main

import (
    "errors"
    "fmt"
)

// 다중 반환값 — Go의 핵심 패턴
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, errors.New("0으로 나눌 수 없습니다")
    }
    return a / b, nil
}

// 가변 인자 (variadic)
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

// 클로저
func counter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

func main() {
    result, err := divide(10, 3)
    if err != nil {
        fmt.Println("에러:", err)
    } else {
        fmt.Printf("결과: %.4f\n", result)
    }

    fmt.Println(sum(1, 2, 3, 4, 5)) // 15

    c := counter()
    fmt.Println(c(), c(), c()) // 1 2 3
}

구조체와 인터페이스

oop.go 
package main

import (
    "fmt"
    "math"
)

// 인터페이스 정의
type Shape interface {
    Area() float64
    Perimeter() float64
}

// 구조체
type Circle struct {
    Radius float64
}

type Rectangle struct {
    Width, Height float64
}

// 메서드 구현 (포인터 리시버)
func (c *Circle) Area() float64 {
    return math.Pi * c.Radius * c.Radius
}
func (c *Circle) Perimeter() float64 {
    return 2 * math.Pi * c.Radius
}
func (r *Rectangle) Area() float64 {
    return r.Width * r.Height
}
func (r *Rectangle) Perimeter() float64 {
    return 2 * (r.Width + r.Height)
}

// 인터페이스를 파라미터로 받는 함수
func printShape(s Shape) {
    fmt.Printf("넓이: %.2f, 둘레: %.2f\n", s.Area(), s.Perimeter())
}

func main() {
    shapes := []Shape{
        &Circle{Radius: 5},
        &Rectangle{Width: 4, Height: 6},
    }
    for _, s := range shapes {
        printShape(s)
    }
}

에러 처리와 defer

errors.go 
package main

import (
    "fmt"
    "os"
)

func readFile(path string) (string, error) {
    f, err := os.Open(path)
    if err != nil {
        return "", fmt.Errorf("파일 열기 실패: %w", err) // %w로 에러 래핑
    }
    defer f.Close() // 함수 종료 시 자동 실행 — 리소스 정리에 필수

    buf := make([]byte, 1024)
    n, err := f.Read(buf)
    if err != nil {
        return "", err
    }
    return string(buf[:n]), nil
}

func main() {
    // defer는 LIFO 순서로 실행
    defer fmt.Println("3: 마지막")
    defer fmt.Println("2: 중간")
    defer fmt.Println("1: 처음")
    fmt.Println("실행 중...")

    content, err := readFile("test.txt")
    if err != nil {
        fmt.Println("에러:", err)
        return
    }
    fmt.Println(content)
}
⚠️ Go 에러 처리 철학

Go는 예외(exception) 대신 에러를 값으로 반환합니다. if err != nil 패턴이 반복적으로 보이는 건 의도적인 설계입니다 — 에러를 명시적으로 처리하도록 강제해 숨겨진 제어 흐름을 없앱니다.

04 — Concurrency

동시성 — 고루틴과 채널

고루틴 기본

goroutine.go 
package main

import (
    "fmt"
    "sync"
)

func worker(id int, wg *sync.WaitGroup) {
    defer wg.Done() // 완료 시 카운터 감소
    fmt.Printf("Worker %d 시작\n", id)
    // 실제로는 여기서 무거운 작업 수행
    fmt.Printf("Worker %d 완료\n", id)
}

func main() {
    var wg sync.WaitGroup

    for i := 1; i <= 5; i++ {
        wg.Add(1)
        go worker(i, &wg) // go 키워드 하나로 고루틴 시작
    }

    wg.Wait() // 모든 고루틴 완료 대기
    fmt.Println("모든 작업 완료")
}

채널을 이용한 통신

channels.go 
package main

import "fmt"

// 생산자: 채널에 데이터 전송
func generate(nums ...int) <-chan int {
    out := make(chan int)
    go func() {
        for _, n := range nums {
            out <- n // 채널로 전송
        }
        close(out) // 전송 완료 시 닫기
    }()
    return out
}

// 소비자: 채널에서 데이터 수신하여 제곱
func square(in <-chan int) <-chan int {
    out := make(chan int)
    go func() {
        for n := range in { // 채널이 닫힐 때까지 수신
            out <- n * n
        }
        close(out)
    }()
    return out
}

func main() {
    // 파이프라인 패턴
    c1 := generate(2, 3, 4, 5)
    c2 := square(c1)
    for n := range c2 {
        fmt.Println(n) // 4, 9, 16, 25
    }

    // select — 복수 채널 동시 대기
    ch1 := make(chan string, 1)
    ch2 := make(chan string, 1)
    ch1 <- "from ch1"
    ch2 <- "from ch2"

    select {
    case msg := <-ch1:
        fmt.Println(msg)
    case msg := <-ch2:
        fmt.Println(msg)
    }
}

05 — FFI with C

C 언어와 CGo FFI

CGo는 Go 코드에서 C 코드를 직접 호출할 수 있게 해주는 Go의 내장 외부 함수 인터페이스(FFI)입니다. C 라이브러리 재사용, 성능 임계 코드 최적화, OS 시스템 콜 직접 접근 등에 활용됩니다.

기본 CGo 사용법

basic_cgo.go 
package main

/*
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

// C 함수 정의 (Go 파일 안에 직접 작성 가능)
int add(int a, int b) {
    return a + b;
}

void greet(const char* name) {
    printf("Hello from C, %s!\n", name);
}
*/
import "C" // 주석 바로 아래에 위치해야 함 (빈 줄 없이)

import (
    "fmt"
    "unsafe"
)

func main() {
    // C 함수 직접 호출
    result := C.add(10, 20)
    fmt.Println("C add 결과:", int(result))

    // Go 문자열 → C 문자열 변환
    name := "Gopher"
    cName := C.CString(name)       // C 메모리에 할당
    defer C.free(unsafe.Pointer(cName)) // 반드시 해제!
    C.greet(cName)

    // C 문자열 → Go 문자열 변환
    cStr := C.CString("Hello from C")
    defer C.free(unsafe.Pointer(cStr))
    goStr := C.GoString(cStr)
    fmt.Println(goStr)
}

외부 C 라이브러리 연동

libmath.go 
package main

/*
#cgo LDFLAGS: -lm        // 링커 플래그: libm 수학 라이브러리 연결
#cgo CFLAGS: -O2 -Wall   // 컴파일러 플래그

#include <math.h>

double fast_sqrt(double x) {
    return sqrt(x);
}

// 구조체 사용 예시
typedef struct {
    double x;
    double y;
} Point;

double distance(Point a, Point b) {
    double dx = a.x - b.x;
    double dy = a.y - b.y;
    return sqrt(dx*dx + dy*dy);
}
*/
import "C"

import "fmt"

func main() {
    // C 함수로 제곱근 계산
    val := C.fast_sqrt(144.0)
    fmt.Printf("sqrt(144) = %.1f\n", float64(val)) // 12.0

    // C 구조체 생성 및 사용
    p1 := C.Point{x: 0, y: 0}
    p2 := C.Point{x: 3, y: 4}
    dist := C.distance(p1, p2)
    fmt.Printf("두 점 사이 거리: %.1f\n", float64(dist)) // 5.0
}

CGo에서 Go 함수를 C에서 호출 (역방향)

callback.go 
package main

/*
#include <stdio.h>

// export 주석으로 표시한 Go 함수를 C에서 호출 가능
extern int goMultiply(int a, int b);

void callGoFromC(int x, int y) {
    int result = goMultiply(x, y);
    printf("C가 Go 함수 호출 결과: %d\n", result);
}
*/
import "C"
import "fmt"

//export goMultiply
func goMultiply(a, b C.int) C.int {
    fmt.Printf("Go: %d × %d 계산 중\n", int(a), int(b))
    return C.int(int(a) * int(b))
}

func main() {
    C.callGoFromC(6, 7) // C → Go → C 순서로 호출
}
⚠️ CGo 주의사항

CGo를 사용하면 크로스 컴파일이 복잡해지고 빌드 속도가 느려집니다. C 메모리(C.CString 등)는 반드시 C.free()로 수동 해제해야 합니다.

🔄 타입 변환 치트시트

C.int ↔ Go int32
C.long ↔ Go int64
C.CString() : Go→C 문자열
C.GoString() : C→Go 문자열

📌 CGo 없는 대안

순수 Go에서 C 라이브러리를 공유 라이브러리(.so)로 불러오는 syscall.LoadLibrary 또는 purego 라이브러리를 통해 CGo 없이도 FFI가 가능합니다.

CGo 빌드 방법

terminal 
# 일반 빌드 (CGo 자동 활성화)
$ go build .

# CGo 비활성화 (순수 Go 바이너리)
$ CGO_ENABLED=0 go build .

# Linux → Windows 크로스 컴파일 (CGo 사용 시 복잡)
$ GOOS=windows GOARCH=amd64 CGO_ENABLED=0 go build .

# CGo 사용 시 크로스 컴파일은 별도 크로스 컴파일러 필요
$ CC=x86_64-w64-mingw32-gcc GOOS=windows CGO_ENABLED=1 go build .
